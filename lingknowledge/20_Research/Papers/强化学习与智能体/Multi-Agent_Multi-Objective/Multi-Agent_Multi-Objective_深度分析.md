# 多智能体系统求解多目标约束优化（深度分析）

> 关联：[[2026-06-20论文推荐]] ｜ [arXiv](http://arxiv.org/abs/2606.20236v1) ｜ [PDF](https://arxiv.org/pdf/2606.20236v1)

## 一、研究问题

计算与网络系统中的许多决策问题，可自然地表述为 **"性能约束下的成本最小化"**。在动态环境中，常用强化学习（RL）在运行时求解：把成本与约束违反通过 **加权惩罚项** 揉进一个标量奖励（拉格朗日式 / Lagrangian-inspired）。

**痛点**：学到的策略行为 **高度依赖这些权重的选择**，而权重通常靠 **人工手调**。这导致：
- 难以找到既满足约束又成本最优的权重；
- 权重稍变，策略行为剧烈漂移，调参昂贵且不稳定。

## 二、方法概述：双智能体架构（MAMO）

论文提出用 **两个智能体** 解耦"权重选择"与"任务执行"：

![[MAMO.png|700]]

- **WA agent（Weight-Assignment Agent，权重分配智能体）**：负责选择满足 **Σₘ wₘ = 1** 的权重向量 (w₁,…,w_M)，决定各目标/约束在标量奖励中的相对重要性。
- **TE agent（Task-Execution Agent，任务执行智能体）**：在环境中以 **加权奖励 = 主成本 + 约束违反惩罚** 进行交互（观察 state，输出 action）。
- **上层反馈回路**：环境返回 **聚合性能指标**（主目标值、约束违反量），反馈给 WA agent，使其自适应地调整权重——把"人工调权"升级为"可学习的上层决策"。

## 三、应用场景：边缘 / 无服务器函数部署

论文的具体落点之一是 **边缘节点的函数副本（function replica）调度**：

![[scaling2.png|700]]

- 在 **资源受限的边缘节点** 上，预启动副本会产生成本（Starting Replicas Incurs Cost），但副本不足又会因 **冷启动（Cold Start）** 或 **系统拒绝（System Rejection）** 损害服务质量。
- 目标：在 **动态负载** 下平衡 **计算成本** 与 **服务质量（QoS）**，并控制 **拒绝概率（Rejection Probability）** 在容忍范围内。
- 这正是一个典型的"多目标 + 约束"问题，验证了 MAMO 框架的实用性。

## 四、关键创新

- **权重学习的智能体化**：把拉格朗日约束 RL 中最棘手的"惩罚权重人工调参"转化为一个 **上层 RL 智能体的决策问题**，实现自适应。
- **职责解耦**：WA（决定"在意什么"）与 TE（决定"怎么做"）分离，降低耦合、便于分别优化。
- **闭环聚合反馈**：用主目标值 + 约束违反量作为上层奖励信号，形成自调节回路。

## 五、深度分析与评估

**亮点**
- 直击约束 RL 的真实工程痛点（权重敏感、人工调参），思路干净、可迁移性强。
- 边缘 serverless 调度场景具体、可落地，便于复现与扩展。

**局限**
- 双智能体引入额外训练复杂度与样本成本；上层 WA agent 的收敛性/稳定性需关注。
