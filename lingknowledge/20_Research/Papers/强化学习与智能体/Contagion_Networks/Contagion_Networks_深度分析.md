# Contagion Networks：多智能体 LLM 系统中的评估偏差传播（深度分析）

> 关联：[[2026-06-20论文推荐]] ｜ [arXiv](http://arxiv.org/abs/2606.20493v1) ｜ [PDF](https://arxiv.org/pdf/2606.20493v1)

## 一、研究问题

当 LLM 被用作多智能体系统中的 **评估者（evaluator / LLM-as-judge）** 时，它们自身的 **系统性评估偏差** 会不会通过 Agent 之间的交互 **传播、放大**？此前对 LLM 评估偏差的研究多停留在单模型层面，缺乏对"偏差在 Agent 网络中如何扩散"的形式化刻画。

## 二、方法概述：传染网络框架

论文提出 **Contagion Networks（传染网络）**，一个量化评估偏差在交互 Agent 网络中传播的形式化框架。

![[fig1_contagion_network_page1.png|620]]

- 核心量是 **Cross-Agent Contagion Matrix（跨 Agent 传染矩阵）Γ₃**，其元素 γ 表示一个 Agent 的偏差向另一个 Agent 传播的强度。
- 用矩阵的 **谱半径 ρ(Γ₃)** 判断系统处于 **放大区（ρ>1）** 还是 **抑制区（ρ<1）**。
- 在 **DeepSeek-chat** 上设置 3 个 Agent，赋予三种不同的评估者偏差画像：
  - **Structured（结构化）**
  - **Balanced（平衡）**
  - **Evidence-based（证据型）**

## 三、关键发现

**1）偏差确实会在 Agent 间稳定传播**

即使三个 Agent 底层是 **同一个模型**，评估偏差仍持续地相互传染，传染系数 **γ ∈ [0.157, 0.352]**（见上图各边权重）。

**2）拓扑结构决定系统命运**

- **全连接拓扑**：ρ(Γ₃) = **1.402 > 1** → 处于 **放大区**，偏差自我强化、越传越强。
- **链式拓扑**：所有 γ < 1.0 → 处于 **抑制区**，偏差沿链条衰减。

![[fig4_chain_propagation_page1.png|680]]

如上图（3-hop 链式传播）所示：GPT-4o→DeepSeek（γ=0.254）、DeepSeek→Claude（γ=0.113）、Claude→GPT-4o（γ=0.191），单跳系数均 <1，累积 β₃≈0.005，三跳后几乎完全衰减（放大约 0.0×）。即 **链式结构天然抑制偏差扩散**。

**3）缓解策略**

论文进一步给出缓解视角（见仓库内 `images/fig3_mitigation_page1.png`），核心思想即通过 **改变 Agent 通信拓扑**（从全连接转向链式/稀疏）把系统从放大区推入抑制区。

## 四、深度分析与评估

**亮点**
- **概念新颖**：把流行病学的"传染 / 谱半径"语言迁移到多智能体 LLM 评估，给出可计算的判据（ρ≷1）。
- **强可操作性**：拓扑选择直接影响偏差是否放大，对真实多 Agent 系统（评审、辩论、自我反思链）设计有直接指导。
- **图示清晰**：传染网络图 + 谱半径 + 链式衰减条形图，结论一目了然。

**局限**
- 实验规模小（3 Agent、受控偏差画像），真实系统中 Agent 数量更多、偏差更复杂。
- γ 的测量依赖偏差画像的人为设定，外推到自然涌现偏差需谨慎。
- 单作者短文，结论需更大规模复现。

## 五、启示

- 设计多智能体评估/协作系统时，**通信拓扑是一个被低估的安全旋钮**：全连接虽利于信息共享，却可能放大系统性偏差；链式/稀疏拓扑更利于抑制偏差扩散。
- 与 [[2026-06-20论文推荐]] 中第 3 篇（多智能体多目标优化）一起看，可见多智能体研究正从"能力叠加"转向"系统动力学与风险控制"。
